## ADDED Requirements

### Requirement: File path uniqueness per version
The system SHALL ensure file paths are unique within each version.

#### Scenario: Unique file path per version
- **WHEN** storing files for a version
- **THEN** each (version_id, path) combination is unique
- **AND** duplicate paths in same version are rejected

#### Scenario: Same path across versions
- **WHEN** same file path exists in different versions
- **THEN** both versions can have the file with different content
- **AND** uniqueness is scoped to single version

### Requirement: File source tracking
The system SHALL categorize files by their origin using file_source enum.

#### Scenario: Template files
- **WHEN** file comes from template base
- **THEN** file_source is set to 'template'
- **AND** indicates file should not be modified

#### Scenario: AI-generated files
- **WHEN** file is generated by Claude
- **THEN** file_source is set to 'ai_generated'
- **AND** indicates AI-created but not yet modified

#### Scenario: Modified files
- **WHEN** file has been changed via refine workflow
- **THEN** file_source is set to 'modified'
- **AND** indicates file differs from AI-generated version

### Requirement: File content storage
The system SHALL store complete file contents as TEXT in database.

#### Scenario: Store code files
- **WHEN** saving TypeScript, JavaScript, or other code files
- **THEN** full file content is stored in content column (TEXT type)
- **AND** content can be retrieved exactly as stored

#### Scenario: Store configuration files
- **WHEN** saving JSON, YAML, or config files
- **THEN** full file content is stored as-is
- **AND** preserves formatting and structure

### Requirement: File modification tracking
The system SHALL track when each file was last modified.

#### Scenario: File update timestamp
- **WHEN** file content is updated
- **THEN** updated_at is set to current timestamp
- **AND** timestamp reflects most recent modification

### Requirement: File cascade relationships
The system SHALL delete files when project or version is deleted.

#### Scenario: Delete project removes files
- **WHEN** project is deleted
- **THEN** all files across all versions are deleted

#### Scenario: Delete version removes files
- **WHEN** version is deleted
- **THEN** only files for that version are deleted

### Requirement: Efficient file queries
The system SHALL provide indexed access to files by project_id and version_id.

#### Scenario: Query project files
- **WHEN** fetching all files for a project (across all versions)
- **THEN** database uses index on project_id

#### Scenario: Query version files
- **WHEN** fetching files for specific version
- **THEN** database uses index on version_id
