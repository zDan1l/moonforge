## Decisions

### 1. Template Storage Strategy

**Decision:** Template files stored in `packages/generator/src/templates/` as regular files.

**Rationale:**
- Simple untuk maintenance dan version control
- Template statis, tidak perlu database storage
- Mudah di-inspect dan debug
- Sesuai dengan struktur MoonForge monorepo (code first)

**Structure:**
```
templates/
├── backend/       # Hono.js API template
├── frontend/      # TanStack Start frontend template
├── root/          # Root files (package.json, pnpm-workspace.yaml, etc)
└── packages/      # Shared packages template
    └── types/    # Empty structure, to be filled by AI
```

### 2. File Source Tracking

**Decision:** Setiap file ditandai dengan `file_source` attribute.

**Rationale:**
- Consistent dengan database schema (`project_files.file_source`)
- Diperlukan untuk UI labeling (template vs AI vs modified)
- Akan berguna untuk Refinement Flow

**Values:**
- `template` - Direct copy from template
- `ai_generated` - Generated by AI (initial or refined)
- `modified` - Changed by user via refine

### 3. Template Merger Strategy

**Decision:** Layer-based merger - template sebagai base, AI content menimpa/overlap.

**Rationale:**
- Template adalah foundation yang sudah teruji
- AI content bersifat spesifik per project
- Perlu handle conflict resolution (AI menang atas template di area overlap)

**Implementation:**
```typescript
interface GeneratedFile {
  path: string;
  content: string;
  source: 'template' | 'ai_generated' | 'modified';
}

function mergeTemplateWithAI(
  templateFiles: GeneratedFile[],
  aiFiles: GeneratedFile[]
): GeneratedFile[] {
  const result = new Map<string, GeneratedFile>();

  // 1. Add all template files
  for (const file of templateFiles) {
    result.set(file.path, { ...file, source: 'template' });
  }

  // 2. Add/override with AI files
  for (const file of aiFiles) {
    result.set(file.path, { ...file, source: 'ai_generated' });
  }

  return Array.from(result.values());
}
```

### 4. Zip Creation

**Decision:** Use `adm-zip` library untuk zip creation.

**Rationale:**
- Native implementation, no external dependencies
- Built-in Node.js support
- Better performance for large files
- Consistent with Moon tech stack opinionated

**Alternative considered:** `jszip` - lebih populer tapi butuh external dependency

### 5. File Structure Builder

**Decision:** Recursive folder creation dengan path validation.

**Rationale:**
- MoonForge structure fixed (TBD by PRD)
- Need handle both files dan folders
- Path validation untuk security

**Output Structure:**
```
{project_name}/
├── .moon/
├── apps/
│   ├── platform/   [T]
│   ├── admin/      [T]
│   └── api/        [T + AI]
├── packages/
│   └── types/      [AI]
├── .env.example
├── package.json
└── pnpm-workspace.yaml
```

### 6. Mock AI Content Generator

**Decision:** Hardcoded template strings untuk MVP testing.

**Rationale:**
- Memungkinkan implementasi tanpa tunggu AI Integration
- Memvalidasi alur generator dari awal sampai akhir
- Mock content akan diganti nyaman setelah AI Integration spec selesai

**Mock Data:**
- Simple Prisma schema (2-3 model basic)
- Empty modules (routes.ts, schema.ts, service.ts skeleton)
- Basic types export

## Risks / Trade-offs

### Risk: Template Updates

**Risk:** Template files statis, sulit update jika ada perubahan.

**Mitigation:**
- Template berupa koding, mudah di-commit ke git
- Untuk MVP, template cukup stabil
- Post-MVP: bisa tambahin template management system

### Risk: Large Zip Files

**Risk:** Generate zip mungkin lambat untuk project besar.

**Mitigation:**
- Streaming zip creation (bukan load semua ke memory)
- Limit project size untuk MVP
- Post-MVP: background processing

### Trade-off: Mock AI Content

**Decision:** Mock content dulu, real AI nanti.

**Implication:**
- Tidak bisa demo full capability di MVP
- Tapi memvalidasi alur generator dari awal
- Mock content akan diganti saat AI Integration

## Open Questions

1. **Zip library:** Gunakan `adm-zip` atau alternatif lain?
   → **Default:** adm-zip (native, no external deps)

2. **Template updates:** Bagaimana handle jika template perlu diupdate?
   → **Default:** Manual git commit untuk MVP, otomatis post-MVP

3. **File conflict:** Bagaimana handle jika AI generate file dengan path yang sama tapi isi beda?
   → **Default:** AI menang (overwrite), versioning handled oleh project_versions

4. **Package.json location:** Di root atau per-app?
   → **Default:** Paket JSON per-app + root (sesuai Moon structure)
