## 📋 QUICK SETUP CHEATSHEET

### One-Liner Setup
```bash
git clone <repo> && cd moonforge && cp .env.example .env && \
docker-compose -f docker-compose.dev.yaml up -d postgres && \
pnpm install && cd apps/api && pnpm prisma migrate dev --name init && \
cd ../.. && pnpm run dev
```

### Setup Time Breakdown
```
Phase 1 (Env Setup):       5 min
Phase 2 (Config):          5 min
Phase 3 (Database):       10 min
Phase 4 (Dependencies):   10 min
Phase 5 (Prisma):         10 min
Phase 6 (Code Quality):    5 min
Phase 7-10 (Verification): 10 min
────────────────────────────────
TOTAL:                    ~45 min
```

### Commands You'll Use Most

```bash
# Daily development
pnpm run dev          # Start all services

# Code quality
pnpm lint            # Check code
pnpm format          # Fix formatting
pnpm type-check      # Check types

# Database
pnpm db:migrate      # Run migrations
pnpm db:push         # Push schema
pnpm db:seed         # Seed data

# Building
pnpm build           # Build all packages

# Monorepo
pnpm --filter <app> run <script>  # Run in specific workspace
```

## 🗄️ DATABASE TABLES QUICK REFERENCE

### users
```
id (UUID)          Primary key
email (VARCHAR)    Unique, login identifier
name (VARCHAR)     Display name
created_at         Account creation timestamp
```

### projects
```
id (UUID)          Primary key
user_id (UUID)     Foreign key to users
name (VARCHAR)     Project name
description (TEXT) User's original prompt
status (ENUM)      draft | generated | refined
created_at         Timestamp
updated_at         Timestamp
```

### project_versions
```
id (UUID)          Primary key
project_id (UUID)  Foreign key to projects
version_number (INT) Auto-increment per project
label (VARCHAR)    Optional version label
created_at         Timestamp
```

### project_files
```
id (UUID)          Primary key
project_id (UUID)  Foreign key to projects
version_id (UUID)  Foreign key to project_versions
path (VARCHAR)     File path (e.g., "apps/api/src/index.ts")
content (TEXT)     Full file content
file_source (ENUM) template | ai_generated | modified
updated_at         Timestamp
```

### chat_messages
```
id (UUID)          Primary key
project_id (UUID)  Foreign key to projects
version_id (UUID)  Foreign key to project_versions
role (ENUM)        user | assistant
content (TEXT)     Message content
file_changes (JSON) List of changed file paths
created_at         Timestamp
```

## 🔄 Git Workflow Setup

### Initialize Repository

```bash
# Clone and setup git
git clone <repo>
cd moonforge

# Configure git hooks with Husky
pnpm husky install

# Create initial commit
git add .
git commit -m "Initial commit: setup environment"
```

### Commit Conventions

Use conventional commits:
```
feat: Add new feature
fix: Fix a bug
docs: Documentation changes
style: Code style changes (formatting, missing semicolons, etc)
refactor: Code refactoring without feature changes
test: Adding or updating tests
chore: Build process, dependencies, tooling changes
```

Example:
```bash
git commit -m "feat(api): add users module with CRUD endpoints"
git commit -m "fix(prisma): correct schema relationships"
git commit -m "docs: update setup guide"
```

## 🔍 Debugging Tips

### TypeScript Errors

```bash
# Clear and regenerate Prisma types
cd apps/api
rm -rf node_modules/.prisma/client
pnpm prisma generate

# In IDE, restart TypeScript server
# (Ctrl+Shift+P → "TypeScript: Restart TS Server")
```

### Database Issues

```bash
# Check database connection
psql postgresql://moonforge:moonforge@localhost:5432/moonforge

# View all tables
\dt

# Drop and recreate (CAUTION: Dev only!)
cd apps/api
pnpm prisma migrate reset
```

### Port Conflicts

```bash
# Find what's using port 3000
lsof -i :3000

# Kill the process
kill -9 <PID>

# Or change port in docker-compose.yml
```

### Module Resolution Issues

```bash
# Clear caches
pnpm store prune
rm -rf node_modules
pnpm install

# Ensure workspaces are linked
pnpm ls -r
```

## 📈 Performance Optimization Tips

### Frontend

```typescript
// Code splitting with TanStack Router
import { lazy, Suspense } from 'react';

const Dashboard = lazy(() => import('./routes/dashboard'));

<Suspense fallback={<Loading />}>
  <Dashboard />
</Suspense>
```

### Backend

```typescript
// Database connection pooling
const prisma = new PrismaClient({
  log: process.env.NODE_ENV === 'development' ? ['query'] : []
});

// Query optimization
const users = await prisma.user.findMany({
  take: 10, // Pagination
  skip: 0,
  include: { projects: true } // Eager load
});
```

### Caching Strategy

```typescript
// In-memory cache with invalidation
const cache = new Map();

function getCached(key, fn) {
  if (cache.has(key)) return cache.get(key);
  const value = fn();
  cache.set(key, value);
  return value;
}

function invalidate(key) {
  cache.delete(key);
}
```
